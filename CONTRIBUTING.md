# Contributing to RTP Supply Chain Model

Thank you for your interest in contributing to this research project. We welcome contributions from the academic community and industry practitioners.

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs or suggest enhancements
- Provide detailed descriptions including steps to reproduce any bugs
- Include relevant system information and error messages

### Submitting Changes

1. **Fork the repository** and create a new branch for your contribution
2. **Make your changes** with clear, descriptive commit messages
3. **Test your changes** to ensure they don't break existing functionality
4. **Submit a pull request** with a clear description of your changes

### Code Style

- Follow PEP 8 guidelines for Python code
- Include docstrings for new functions and classes
- Add comments for complex logic

## Academic Collaboration

### Research Contributions

We encourage academic contributions including:

- **Model extensions**: New constraints, node types, or supply chain configurations
- **Validation studies**: Empirical validation with real-world data
- **Comparative analyses**: Benchmarking against other models or approaches
- **Documentation**: Improved explanations, tutorials, or case studies

### Citation

If you use this model in your research, please cite our work. Citation information will be provided upon publication.

### Data Sharing

If you have relevant datasets that could help validate or improve the model, please contact us to discuss potential collaboration.

## Academic Code of Conduct

- **Integrity**: Ensure all contributions are original or properly attributed
- **Reproducibility**: Document methods and parameters to enable replication
- **Transparency**: Disclose any conflicts of interest or funding sources
- **Respect**: Engage constructively with other contributors and maintainers

## Getting Help

For questions about contributing:

- Open a GitHub Issue for technical questions
- Contact the maintainers directly for collaboration inquiries (see below)
